MPWebServices--Help file


Tab: WebServer

Http Port:
Set the http port where this wepapp should be available. If any other application is also listening on this port, starting the server will fail

Player Type:
Sets the type how content will be played back when streamed. Choose between the following 3 methods
  • Player installed on client When clicking the play button you receive a .bat file which starts the player configured in Client Streaming Player and starts playback.
  • VLC Browser Plugin Plays the media file in special webpage. VLC Browser plugin is required on client side.
  • .m3u playlist Gives you an .m3u playlist file which can be played back with any player that is associated with .m3u files. This is probably the most flexible and preferred method

Client streaming player:
You only need to fill in something here if you selected the Player installed on client player type. Just set the full path to the executable here. This has to be the same for all clients.

Username/Password
Configure the authentication credentials to access the web app

Streaming URL
Here you can override the streaming URL. This is only necessary if you access these pages from the internet. If you use e.g. something like dyndns.org you should set your dyndns name here

Tab: Thumbnails/Logos


Thumb width/height
The dimensions of the thumbnails in pixels

Tv / Radio / Pictures
Configure the paths to your tv/radio logo directories and to your picture thumbnails

Tab: MP DB locations


Here you can configure the full path and filename to some mp databases to be able to access them from the web app

Tab: Transcoding


In this section you can configure various streaming profiles.
Each profile consists of the following fields:
  • Name Just a descriptive name
  • Transcode If unchecked, all other fields are ignored and the content won't be transcoded. Otherwise you have to fill out the other fields
  • Transcoder Full path and filename of the transcoder e.g. mencoder on the server pc
  • Params The command line for the transcoder where {0} is a placeholder for the input and {1} is a placeholder for the output
  • Input/Outputmethod These are the available methods you can use for the {0} and {1} parameters not all methods are supported by every transcoder. e.g. inputmethod=namedpipe and output=namedpipe works good with mencoder but also input=filename will work. Just play around with these settings ;-)
